Príprava projektu

Pri rozdelení úloh som si vybral klientskú časť. S tímom sme sa ju rozhodli implementovať webovou aplikáciou (single page aplication).

Mal som skúsenosti s Reactom a Reduxom z minulosti a tieto technológie sa na tento projekt hodili. Navyše sme sa dohodli na dizajne, ktorý je Gentelella admin template, postavený na bootstrape.

Veľkú časť pribalených knihovní som nepoužil, pretože boli pre túto aplicáciu zbytočné (napríklad počasie, profil, prihlasovanie). Ponechal som ale ChartJS, JSVectormap a NProgress.

Jan vytvoril mock api na apiary.io a mohol som začať vyvíjať frontend. Postupne som pridal potrebné balíčky, ich zoznam sa dá nájsť v package.json

Návrh frontendu

Hlavná myšlienka bola intuitívnosť pre užívateľa. Frontend si vyžiada len tie dáta, ktoré v danom okamihu potrebuje.

Aplikácia je rozdelená na komponenty, stav aplikácie sa uchováva v redux store. Na asynchrónne api volania som použil redux-thunk. Veľká časť store je tvorená Immutable objektami.

Aplikácia podporuje React a Redux vývojárske prídavky do Chrome.

Implementácia frontendu

Najskôr som implementoval základnú funkcionalitu pomocou apiary mocku. Následne som počkal na implementáciu api a dokončil funkčný frontend.

Výsledká aplikácia sa dá spustiť vo vývojárskom móde npm run dev alebo skompilovať pomocou npm run production a výstup sa nachádza v /dist priečinku.

Dokončenie projektu

Po dokončení sme sa zišli s tímom a upravili pár maličkostí a vytvorili prezentáciu.
